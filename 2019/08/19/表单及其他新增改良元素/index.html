表单及其他新增改良元素

发表于 2019-08-19 更新于 2019-08-20 分类于 编程 ， 前端 Valine：

新增元素和属性

表单内元素的form属性

表单元素可以写在任何位置，只要为其定义一个form属性，那么他就属于某个表单

formaction属性

为不同的提交按钮增加不同的formaction属性,跳转到不同的页面

formmethod属性

为不同的提交按钮设定不同的提交方式

formenctype属性

为不同的提交按钮设定不同的数据编码方式

属性值 说明
application/x-www-form-urlencoded 默认提交编码
multipart/form-data 不对字符编码；使用包含文件上传的表单，必须使用该值
text/plain 表单数据中的空格被转换为+号，不对特殊字符进行编码

formtarget属性

为不同的按钮指定在何处打开加载的页面

属性值 说明
_blank 在新的浏览器窗口打开
_self 默认值，在当前框架打开
_top 在当前浏览器窗口打开
_parent 在当前框架的父框架中打开

autofocus属性

当前页面打开时，自动获取焦点

required属性

指定该属性后，如果元素内容为空（空格可以），不允许提交，同时浏览器现实提示文字

labels属性

为所有可使用labe标签的元素指定一个labels属性(script脚本中调用)，代表该元素所绑定的所有nodelist元素

control属性

通过该标签的dom元素的control元素来访问该标签所绑定的dom表单元素

placeholder属性

未输入状态的提示

设定提示文字的样式

list属性

单行文本框定义的属性，与datalist元素一起使用 值为datalist元素的id.

为了避免在没有支持该元素的浏览器上出现错误，通常将datalist元素设置display: none

autocomplete属性

自动完成 on和off属性

pattern属性

属性值为正则表达式，用于表单验证。模式不匹配，会提示

selectionDirection属性

针对input元素和textarea元素新增的属性（js调用）,当用户在这两个元素中选取部分文字时，可以使用该属性获取选取方向。正向选取，该属性为forward；反向选取，该属性为backward。默认是forward

代码 作用
document.forms 获取当前页面的所有表单
docume.forms[0] 获取当前页面的第一个表单
document.forms[‘exportServlet’] 获取当前页面name为exportServlet的表单
document.forms[0].submit 提交当前页面的第一个表单

复选框的indeterminate属性

checkbox中设置indeterminate属性，表示复选框处于”尚未明确是否选取”状态（js使用）

indeterminate与checked属性是两种不同的属性
应该先判断indeterminate 然后再判断checked属性

textarea元素的maxlength与wrap属性

  • maxlength属性 限定textarea元素可输入文字的最大个数
  • wrap属性 指定属性值为soft或者hard。hard在提交表单的时候，会显示换行符（必须加上cols属性）；soft在提交表单的时候，不会显示换行符

其他新增input元素

  • url类型：专门用来输入url地址的文本框
  • email类型：专门用来输入email地址的文本框

该类型具有一个multiple属性，允许在该文本框输入一段以逗号分隔的email地址

  • date类型：专门用来表示日期
  • time类型：专门用来表示时间
  • datetime-local类型：专门用来表示时间日期
  • month类型：专门用来输入月份的
  • week类型：专门用来输入周号的
  • step属性 用户从日历中选择日期时对选择值的限定

  • setpUp 表示向上变更 step表示向下变更

  • min和max属性 表示输入限定值的范围
  • valueAsDate 用来设置或获取UTC日期和时间 valueAsNumber用来设置或获取时间毫秒数
  • number类型：一种专门用来输入数字的文本框 也有min max step属性

valueAsNumber 也可以用来设置获取值

  • range类型：滑动条，也具有min max step属性
  • search类型：搜索关键词的文本框
  • tel类型：输入电话号码的文本框
  • color类型：选取颜色的文本框

表单验证

自动验证

pattern require属性

取消验证

问题：一个非常大的表单分为两部分，现在有个需求，先提交第一部分，然后在提交第二部分。
解决1：form元素有个novalidate属性用来关闭整个表单验证，先把novalidate设置为true，关闭表单验证，提交第一部分内容。然后把novalidate设置为false，打开表单验证，提交第二部分内容。
解决2：input元素有一个formvalidate属性，为true表示关闭此元素的验证。（第二部分的表单元素数量很少时可以使用这个方法）

submit也有formvalidate属性，单击该按钮时，整个表单验证都失效了

validity属性

form元素和input元素的属性，返回一个ValidityState对象，其中有一个valid属性，表示表单内所有内容是否有效或单个input元素内容是否有效

增强的页面元素

figure元素和figcaption元素

figure一种元素的组合，表示网页中一块独立的内容，将其从网页中移除后不会对其他内容产生任何影响。可以是图片、统计图或者代码示例

details和summary元素

details元素用于标志该元素内部可以展开收缩的元素。该元素具有一个open属性，当该属性为true时，子元素应该展开；为false，子元素收缩。summary元素表示details单击时的说明文字（如果没有summary元素，details会提供默认文字），每次收缩或者展开，details都将触发toggle事件

mark元素

表示页面中需要突出显示的文字，目的是为了吸引用户的注意力

mark元素的标识与原文作者无关，只是为了吸引用户的注意力
strong元素是原文作者用来强调一段文字的重要性，比如警示消息、错误信息。
em是为了突出文章重点

progress元素

代表一个任务的完成进度 有value和max属性

meter元素

表示规定范围内的数量值

  • value属性
  • min属性
  • max属性
  • low属性：表示范围的下限值
  • high属性：表示范围的上限值
  • optimum：最佳值

dialog元素

代表一个对话框：默认出于隐藏状态，调用show方法显示；close方法隐藏

showModal方法以模式对话框显示
returnValue属性可以为对话框设置或者返回一个返回值

改良的a元素

download属性 点击即下载 属性值可以是路径，也可以是文件名，代表下载之后的文件的文件名

改良的ol属性

  • start属性：表示ol里面的li元素标号从哪开始
  • reversed属性：表示反向排列

加以严格限制的cite元素

cite表示作品的标题（不能包括作者以及任何人名）

安全性增强的iframe元素

为iframe中新增一个sandbox属性,有如下限制

  • 该页面中的插件被禁用
  • 该页面中的表单被禁止提交
  • 该页面的js代码被禁止执行
  • 该页面的超链接被禁用
  • 该页面被视为一个单独的源（没有该属性，被视为同一个源，虽然可以是不同源），禁止加载来自服务器端的内容，禁止获取cookie或者webstorage的内容
    判断浏览器是否支持iframe的sandbox属性

可以给sandbox指定多个属性（多个属性之间用空格隔开）

  1. allow_forms：允许表单提交
  2. allow-scripts：允许执行js脚本
  3. allow-same-origin：允许加载服务器内容，允许获取cookie或者webstorage内容
  4. allow-top-navigation：允许超链接跳转

增强的script元素

script元素新增async属性和defer属性

  • async：异步加载js文件并执行（加载完立即执行）
  • defer：异步加载js文件，js执行放在dom元素解析完成和所有js文件加载完毕后才按照顺序依次执行。defer执行速度较快
