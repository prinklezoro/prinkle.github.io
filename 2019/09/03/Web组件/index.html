使用template元素定制客户端模板

  • 检测浏览器是否支持template元素
  • template元素中的内容不会被浏览器渲染，必须调用才开始渲染
  • 调用之前，模板中的内容并不看作为页面DOM树的内容。使用document.getElement等方法获取不到模板中的元素
  • 模板不可能被预渲染
  • 模板可以放置在任何位置
  • template.content是一个只读属性，包含了模板的DocumentFragment对象
  • 可以在模板中嵌套模板，但是激活外层模板并不会激活内层模板

自身独立的Shadow DOM组件

Shadow的引入

  • template模板的脆弱性：使用新的id或样式类时，无法了解与页面所使用的现有名称冲突，导致微小错误渐渐增多
  • Shadow DOM组件好处：组件的DOM是独立的；内部定义的css样式是有作用域的，作用域只在该组件生效；可以使用简单的css选择器，不用担心与外部样式冲突
  • 借助Shadow DOM影子树，开发者可以创建局部作用于的DOM树并将该DOM树附加至其他元素上，这一作用域子树称之为影子树，被附着的父元素称之为影子宿主

创建Shadow DOM

  • 指定一个影子宿主节点：var host = document.getEle…
  • 调用元素的attachShadow({mode: ‘open’})，创建影子树，参数为一个对象，mode（true）代表开放此影子树DOM；mode（closed）代表关闭此影子树DOM
  • textarea input img br等元素不能作为宿主元素
  • template.remove方法 移除该模板元素

深入Shadow DOM

  • slot元素 在Shadow DOM中定义，在外部写实际的标签引用该slot 把该slot元素渲染成此标签
  • 在影子树内部使用的样式与外部隔离，因此还可以使用相同id的元素
  • 外部的样式也不能影响影子树内部的元素，但是利用某些属性可以操作
  • :host选择器允许你选择与定义一个Shadow DOM宿主元素的样式
  • 父页面中为宿主元素定义的样式较之在元素中定义的:host样式有更高的优先级
  • 利用:host(选择器)函数可以进一步限定宿主元素的状态，例如:host(:hover)
  • :host-context(选择器)可以根据环境进行主题化，在该页面的宿主元素或者上级元素使用的类样式与host-context的选择器相匹配，则该样式规则中定义的所有样式被应用到宿主元素中

template.content只能使用一次，不能渲染到两个或两个以上的宿主元素中，所有要使用cloneNode(true)方法

  • 分布式节点设置样式：::slotted(选择器)选择组件外部的页面中所有与选择器相匹配的slot节点
  • 没有设置slot属性且没有对应slot元素的外部节点样式不会生效
  • CSS变量创建样式钩子:变量名需要规范

高级主题

  • 尽量不要创建关闭模式下的Shadow DOM。创建闭合影子跟之后，javascript代码将无法访问组件的内部dom
  • slot的slotchange事件，当在主DOM中添加或者移除slot元素时，将触发slotchange事件

自定义元素

  • 使用window.customElements.define(name, obj)方法定义自定义元素
  • 第一个参数是一个字符串，定义在dom文档中的标签名字
    • 自定义元素的名称必须包括短横线便于与常规元素区分
    • define方法不能多次定义同一个元素，会报错
    • 自定义元素不能自我封闭，必须写封闭标签
  • 第二个参数表示继承自HTMLElement的对象 class AppDrawer extends HTMLElement{}

在js代码中创建自定义元素

  • new关键字创建元素

自定义元素响应

  • 自定义元素的特殊生命周期钩子
名称 调用时机
constructor 创建或升级一个元素的实例时调用。用于初始化状态、设置事件侦听器或创建Shadow DOM元素
connectedCallback 元素每次插入到页面中都会调用。用于运行安装代码，例如获取资源或渲染
disconnectedCallback 元素每次从页面中移除时都会调用。
attributeChangedCallback(attrName, oldValue, newValue) 属性添加、移除、更新替换时被调用。第一个参数是被更改的属性名字，第二个参数是原来的值，第三个参数是修改后的值。
adoptedCallback 自定义元素被移入新的文档时被调用 例如：document.adoptNode(el)

attributeChangedCallback 仅当自定义元素的observedAttributes属性值中列出的属性才会收到此回调

  • adoptedNode:从另外一个文档中获取节点。节点下的所有子节点都会获取到；节点及其子节点会在源文档中删除
  • importNode:从另外一个文档中获取节点。节点在源文档中不会删除
  • 可以在connectedCallback函数中，定义自定义元素的属性以及内容
  • 静态属性static get observedAttributes(){ return [‘open’]; } 添加observedAttributes属性，以便可以调用attributeChangedCallback回调函数来观察open属性的变化

自定义元素的属性

  • 元素对象的属性通常会将其值以元素属性的形式映射回DOM。
  • 创建自定义dom元素类时，利用get set方法可以为元素指定属性，达到可以映射的目的

结合Shadow DOM使用

  • 在创建自定义元素的构造函数中，使用attachShadow方法作为影子根节点

slot元素没有定义name值，自动与根节点内部的所有元素匹配
