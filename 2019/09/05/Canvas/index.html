Canvas

发表于 2019-09-05 分类于 编程 ， 前端 Valine：

canvas元素的基础属性

  • 页面中放置canvas标签，并指定高度宽度
  • 获取canvas上下文：canvas.getContext(‘2D’),参数也可以是webGL绘制3D
  • 填充和绘制边框 fill 和 stroke
  • 设定绘图样式：fillStyle属性——填充的样式；strokeStyle属性——边框的样式
  • 绘制矩形：fillRect填充矩形 strokeRect绘制矩形边框 参数：起始横坐标；起始纵坐标；宽度；高度
  • clearRect 将指定矩形区域内的绘图擦除 参数同上

使用路径

绘制圆形

  • 开始创建路径 beginPath()
  • 创建圆形路径 arc(x, y, radius, startAngle, endAngle, anticlockwise)，六个参数：起点横坐标；起点纵坐标；开始角度；结束角度（弧度）；顺时针（false）or逆时针（true）
  • 闭合路径 closePath()
  • fillstyle fill 填充 或者strokestyle stroke边框
  • 绘制椭圆 ellipse(x, y, radiusX, radiusY, tation, startAngle, endAngle, anticlockwise) 八个参数：起点横坐标；起点纵坐标；椭圆的横向半径；椭圆的纵向半径；椭圆的顺时针方向旋转角度(度数)；开始绘制角度；结束绘制角度；逆时针or顺时针

如果不关闭路径，在上一次绘制的路径会接着下一次的绘制不会中断，也就是说上一次绘制的图形会接着后面的绘制路径接着绘制

绘制直线

  • moveTo 将光标移到指定坐标点 参数：x坐标；y坐标
  • lineTo 将绘制一条直线 参数：x坐标；y坐标
  • lineCap 为直线添加线帽
    • butt：默认属性值
    • round：直线圆角帽
    • square：为直线添加正方形线帽
  • lineJoin：指定两条直线交汇时的拐角形状
    • miter：默认属性值，创建尖角拐角
    • round：创建圆角拐角
    • bevel：创建斜角拐角
  • setLineDash：绘制虚线，使用一个参数值为数组，里面存放线段长度和线段与线段之间的距离
    • 数组里面只有一个值时：表示线段长度和线段与线段之间的距离都是这个值
    • 数组里面有两个值时：第一个数值表示线段长度；第二个数值表示线段之间的距离
    • 数组里面有偶数个值时：第奇数个值表示线段长度；第偶数个值表示距离（无线循环）
    • 数组里面有奇数个值时：自动复制数组中数值为偶数倍。例如setLineDash([6, 10, 15])相当于setLineDash([6, 10, 15, 6, 10, 15])
    • 方法返回值为该虚线数组

绘制曲线

  • artTo(x1, y1, x2, y2, radiusX, [radiusY, rotation])使用七个参数，后两个参数可选：起点横坐标；起点纵坐标；终点横坐标；终点纵坐标；圆弧横向半径；圆弧纵向半径；椭圆顺时针方向旋转角度

绘制贝塞尔曲线

绘制二次样条曲线

使用Path2D对象绘制路径

  • 可以使用该对象的各种方法绘制直线、矩形、圆形、椭圆以及曲线
方法 作用
moveTo(x,y) 将光标移动到指定坐标点
lineTo(x,y) 在当前坐标点与参数中指定的直线终点之间绘制一条直线
rect(x,y,w,h) 绘制矩形
arc(x,y,radius,startAngle,endAngle,anticlockwise) 绘制圆形或圆弧
ellipse(x,y,radiusX,radiusY,rotation,startAngle,endAngle) 绘制椭圆或椭圆弧
artTo(x1,y1,x2,y2,radiusX,radiusY,rotation) 绘制圆形曲线
bezierCurveTo(cp1x,cp1y,cp2x,cp2y,x,y) 绘制贝塞尔曲线
quadraticCurveTo(cpx,cpy,x,y) 绘制二次样条曲线
closePath 闭合路径
  • 使用context.fill(path)或者context.stroke(path)绘制图形

closePath方法只是闭合路径，与起点闭合；而不是关闭路径。beginPath方法已经有重新开始路径（断开之前路径）的意思

  • path2 = new Path2D(path1) 将path1绘制的路径复制到path2中
  • path2.addPath(path1) 将path1所代表的路径添加到path2对象中

绘制渐变图形

绘制线性渐变

  • context.createLinearGradient(startX,startY,endX,endY),四个参数：起始横坐标；起始纵坐标；终点横坐标；终点纵坐标。返回一个LinearGradient对象。
  • 设定颜色，g.addColorStop(offset, color) 第一个参数：离开渐变起始点的偏移量（0-1之间）；第二个参数：这个点的颜色。如果要渐变的话，一般需要至少两个此函数

绘制径向渐变

  • context.createRadialGradient(stratX, startY, startRadius, endX, endY, endRadius) 六个参数：起始圆的横坐标；起始圆的纵坐标；起始圆的半径；结束圆的横坐标；结束圆的纵坐标；结束圆的半径
  • 同样需要设定addColorStop方法
  • 左图为小圆向大圆渐变，中图为大院向小圆渐变，右图为相离两个圆的渐变，从图中可以很清楚的看到圆相离或相交时的渐变方式，渐变总是从起始圆开始（包括起始圆内部），朝着结束圆方向，直到遇到结束圆（不包括结束圆内部）。

绘制变形图片

坐标变换

  • canvas画布的最左上角为坐标轴原点，如果要对这个原点使用变换处理，就可以实现图形的变形处理，需要用到以下方法
  • context.translate(x, y) 移动坐标轴原点 x表示向右移，y表示向下移
  • context.scale(x, y) x是水平放大倍数；y是垂直放大倍数，缩放中心点是画布原点
  • context.rotate(angle) 旋转角度，旋转轴为坐标原点，旋转中心点是画布原点

坐标变换研究

  1. translate移动是在原来的基础上移动的

  1. rotate旋转是根据translate之后的坐标原点旋转的，并且也是在原来的基础上加角度（原来设置的旋转角度会影响后面所有的绘制图形）

  1. rotate旋转之后，translate移动的方向也会跟着旋转，并不是按照坐标轴方向，或者说rotate旋转的是坐标轴

给图形绘制阴影

  • shadowOffsetX: 阴影的横向位移量
  • shadowOffsetY：阴影的纵向位移量
  • shadowColor：阴影的颜色
  • shadowBlur：阴影的模糊范围

使用图片

绘制图片

  • context.drawImage(image, x, y) 参数：要绘制的Image对象；在画布中的起始横坐标；在画布中起始纵坐标
  • context.drawImage(image, x, y, w, h) 参数：要绘制的Image对象；在画布中的起始横坐标；在画布中的起始纵坐标；在画布中的宽；在画布中的高
  • context.drawImage(image, x1, y1, w1, h1, x2, y2, w2, h2) 参数：要绘制的Image对象；在源图片上的起始横坐标；在源图片上的起始纵坐标；源图片截取的宽；源图片截取的高；在画布中的起始横坐标；在画布中的起始纵坐标；在画布中的宽度；在画布中的高度

图像平铺

  • context.createPattern(image, type) 参数：要平铺的Image对象；type：no-repeat(不平铺)；repeat-x(横方向平铺)；repeat-y（纵方向平铺）；repeat（全方向平铺）。返回值是一个样式对象，赋值给fillStyle
  • 图像平铺相当于背景图片，要绘制图形才能平铺背景图片

图像裁剪

  • 先设置好裁剪区域，创建好路径。路径创建完成后，调用clip()方法裁剪区域

像素处理

  • context.getImageData(sx, sy, sw, sh) 参数：获取区域的起点横坐标；获取区域的起点纵坐标；获取区域的宽度；获取区域的高度
  • 返回值是一个CanvasPixelArray对象，具有height, width, data属性。data是一个保存像素数据的数组，内容类似于[r1, g1, b1, a1, r2, g2, b2, a2…] 代表第一个像素的红色值、绿色值、蓝色值与透明值。
  • putImageData(imagedata, dx, dy [, dirtyX, dirtyY, dirtyWidth, dirtyHeight])，七个参数：imagedata对象，偏移量x，偏移量y，原图像的偏移量x和y，原图像的宽度和高度。将imagedata对象绘制到画布上

图形 图像的组合和混合

组合图形

  • context.globalCompositeOperation 设置组合方式
type 描述
source-over 默认值，表示新图形覆盖在原图形之上
destination-over 在原有图形之下绘制新图形
source-in 新图形与原有图形做in运算，只显示新图形与原有图形相重叠的部分，其他部分都变成透明
destination-in 只显示原有图形与新图形相重叠的部分，其他都变成透明
source-out 新图形与原有图形做out运算，只显示新图形中与原有图形不重叠的部分，其他部分均变成透明
destination-out 原有图形与新图形做out运算，只显示原有图形与新图形不重叠的部分，其他部分透明化
source-atop 只绘制新图形中与原有图形相重叠的部分以及未被重叠覆盖的原有图形，新图形的其他部分透明化
destination-atop 只绘制原有图形中被新图形覆盖的部分与新图形的其他部分
lighter 原有图形与新图形均绘制，重叠部分做加色处理
xor 只绘制不重叠的部分，重叠部分透明化
copy 只绘制新图形，原有图形未与新图形重叠的部分透明化

混合图像

  • 概念：利用某种数学公式将两幅图像混合在一起
  • 设置globalCompositeOperation属性值
type 描述
normal 正常模式
darken 变暗模式
lighten 变亮模式
multiply 正片叠底模式
screen 滤色模式
coor-burn 颜色加深模式
color-dodge 颜色减淡模式
hard-light 强光模式
soft-light 柔光模式
overlay 叠加模式
difference 差值模式
exclusion 排除模式

绘制文字

  • context.fillText(text, x, y, [maxWidth]) 四个参数：要绘制的文字；绘制文字的起点横坐标；绘制文字的七点纵坐标；显示文字的最大宽度（防止文字溢出）
  • context.strokeText 参数同上
  • 在进行绘制之前，可以对要绘制的文字对象设置属性样式
    • font属性：设置文字字体
    • textAlign属性：设置文字水平对齐方式，start/end/left/right/center
    • textBaseLine属性：设置文字垂直对齐方式，top/hanging/middle/alphabetic/ideographic/bottom。默认值为alphabetic
  • font属性：font-style/font-variant/font-weight/font-size/line-height/font-family 例如 context.font=”italic small-caps bold 12px arial”;

font-variant:small-caps 代表小型大写字母
textAlign start与left效果一样，end与right效果一样，默认是start

  • context.measureText(text) 返回一个TextMetrics对象，该对象的width属性表示使用当前字体后text参数中指定的文字的总文字宽度

补充知识

保存与恢复状态

  • save()方法保存当前的绘画状态，包括坐标原点、变换矩阵以及context的属性值
  • restore()方法恢复到上一次保存的状态中

canvas元素图像转换为data URL

  • canvas.toDataURL(type) 参数 表示要输出数据的MIME类型

canvas元素图像转换为Blob对象

  • canvas.toBlob(callback, [mimeType, [qualityArgument]]) 三个参数：第一个参数是转换成功时所执行的函数；第二个参数可选，表示MIME类型；如果第二个参数是”image/jpeg”或”image/webp”，第三个参数可以指定图像质量，参数值为0-1之间的小数

  • 不传第二个参数 默认是png图像

canvas动画

  • canvas动画的原理就是一个不断擦除、重绘、擦除、重绘的过程。主要实现的函数时setInterval

解码图像

  • 如果canvas元素或者image、video对象过大时，浏览器自动进行的解码过程会消耗较大的CPU资源，这样就必须进行手动解码
  • createImageBitmap(image, sx, sy, sw, sh).then(function(response)) 第一个参数必须指定，为要转码的img元素、video元素、canvas元素、blob元素、ImageData元素等等；另外四个参数分别为复制区域的水平起始坐标、宽高
  • 返回值是一个以一个ImageBitmap对象为肯定结果的Promise对象
