发表于 2019-09-06 分类于 编程 ， 前端 Valine：

video元素和audio元素的基础知识

老页面中播放视频音频的缺点

  • 冗长，代码量多
  • 需要使用Flash插件

H5使用视频或音频

  • video标签里面的文字为在不支持video标签时所显示的替换文字
  • 还可以通过source元素为同一个媒体数据指定多个播放格式与编码方式，以确保浏览器可以从中选择一种自己支持的播放格式进行播放
  • codecs表示所使用的媒体的编码格式
  • 最好不要省略type类型，否则浏览器选择时无法判断自己能不能播放而先行下载一小段视频或音频数据，这样会浪费带宽和时间
  • 各浏览器对视频音频格式的支持
浏览器 支持视频 支持音频
IE9 MP4 MP3
火狐 MP4/WebM/ogg MP3/WAV/Ogg
谷歌 MP4/WebM/ogg MP3/WAV/Ogg
Safari MP4/WebM MP3/WAV

属性

  • src 指定媒体数据的URL地址
  • autoplay 是否在页面加载后自动播放
  • preload 指定视频和音频是否预加载。如果使用预加载那么浏览器会预先缓冲视频或音频数据，可以加快播放速度，这个属性有三个可选的值
    • none：表示不进行预加载
    • metadata：表示只加载媒体的元数据(媒体字节数，第一帧，播放列表，持续时间等)
    • auto：表示预加载全部视频或音频 默认是auto
  • poster（video独有属性）：当视频不可用时向用户展示一幅替换用的图片，属性值为图片路径
    • css样式：object-fit可以设置被替换图片的样式：fill(正好填充video内容宽高，会使图片变形)；contain（填充元素保持宽高比，没有覆盖的用黑边）；cover（保持宽高比完全覆盖盒子）
  • loop：是否循环播放视频或音频
  • controls播放用控制条
  • error属性：在读取、使用媒体数据的时候，正常情况下error属性值为null，但是只要任何时候出现错误，error属性返回一个MediaError对象，该对象的code返回对应的错误状态。在onerror事件中通过this.error获取
    • 1：媒体数据的下载过程被用户中止
    • 2：确定媒体资源可用，但是在下载时出现网络错误，下载被中断
    • 3：媒体资源可用，但是解码时发生错误
    • 4：媒体资源不可用或格式不支持
  • networkState属性：在媒体数据加载过程中读取当前的网络状态，在onprogress事件中用this.networkState获取。有以下值
    • 0：表示初始状态
    • 1：视频/音频是活动的且已经选取资源，但未使用网络
    • 2：媒体数据加载中
    • 3：没有支持的编码格式，不执行加载
  • currentSrc：读取当前播放中的媒体数据的URL地址
  • buffered：返回一个对象，该对象实现TimeRanges接口，已确认浏览器是否已缓存媒体数据，该对象表示一段时间范围，具有一个length属性，通常值为1（不存在就为0）；该对象还有两个方法，start(index)和end(index)表示缓存的开始时间和结束时间。
  • readyState：返回媒体当前播放位置的就绪状态
    • 0：没有获取到媒体的任何信息，当前播放位置没有可以播放的数据
    • 1：音视频就绪
    • 2：当前播放位置有数据可以播放，但没有获取到下一帧的数据，或者当前是播放的最后一帧
    • 3：当前及下一帧的数据可用
    • 4：可用数据足以播放
  • seeking：返回一个布尔值，表示浏览器是否正在请求某一特定播放位置的数据，true表示正在请求数据，false表示已停止请求
  • seekable：返回一个TimeRanges对象，表示请求到的数据范围
  • currentTime：当前播放位置（时间），可读可写，单位为秒
  • startTime：媒体播放的开始时间，通常为0
  • duration：媒体文件总的时间
  • played：返回一个TimeRanges对象，该对象可以读取到媒体文件已播放部分的时间段。开始时间与结束时间（start和end）
  • paused：返回一个布尔值，表示是否处于暂停状态
  • ended：返回一个布尔值，表示是否播放完毕
  • defaultPlaybackRate：读取或修改媒体默认的播放速率
  • playbackRate：读取或修改媒体当前的播放速率
  • volume：读取或修改媒体的播放音量，范围为0到1
  • muted：读取后修改媒体的静音状态，true表示静音，false表示非静音

方法

  • play() 播放媒体，自动将paused属性变为false
  • pause() 暂停播放，自动将paused属性变为true
  • load() 重置载入媒体进行播放，自动将元素的playbackRate属性值变为defaultPlaybackRate属性的值
  • canPlayType()测试浏览器是否支持指定的媒体类型
  • type为MIME类型字符串，返回值有三个可能值：空字符串（浏览器不支持此种媒体类型）；maybe：浏览器可能支持此种媒体类型；probably：确定支持此种媒体类型

事件

事件 描述
loadstart 浏览器开始在网上寻找媒体数据
progress 正在获取媒体数据
suspend 延迟下载，客户端暂停获取媒体数据
abort 客户端主动终止下载
error 获取媒体数据出错
emptied 媒体元素所在网络突然变成未初始化状态两个原因：载入媒体过程发生致命错误；load()重新载入媒体
stalled 浏览器尝试获取媒体数据失败
play 即将开始播放，执行play方法触发，或元素被设置为autoplay开始播放时触发
pause 执行pause方法时触发
loadedmetadata 浏览器获取完毕媒体的时长和字节数
loadeddata 浏览器已加载完毕当前播放位置的媒体数据，准备播放时触发
waiting 播放过程中由于得不到下一帧数据时触发，但很快可以得到下一帧数据
playing 正在播放时触发
canplay 可以播放，但中途可能因为加载而暂停，没有全部缓冲完
canplaythrough 可以播放，已经全部缓冲完毕
seeking seeking属性变为true，浏览器正在请求数据
seeked seeking属性变为false，浏览器停止请求数据
timeupdate 播放位置改变 与playing的区别在于playing只触发一次
ended 播放结束后停止播放
ratechange defaultPlaybackRate或playbackRate被改变时
durationchange 播放时长被改变
volumechange volume或者muted被改变
