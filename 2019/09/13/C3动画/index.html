Transitions 功能

基本使用

  • 通过将元素的某个属性从一个属性值在指定的时间内平滑过渡到另一个属性值来实现动画功能
  • transition: property duration timing-function
  • property代表对哪个属性进行平滑过渡；duration表示在多久时间内完成属性值的平滑过渡；timing-function表示过渡的曲线
  • 分开写分别为 transition-property transition-duration transition-timing-function
  • 还有第四个参数：transition-delay: ns 表示过渡延迟多少秒执行
  • transition-timing-function 有多个值：
    • linear 匀速
    • ease-in 加速
    • ease-out 减速
    • cubic-bezier函数：自定义速度模式
  • 最后一个参数，可以使用工具网站定制

多个属性值同时过渡

  • 分别写每个属性值的过渡样式，用逗号隔开
  • 所有属性一起写，直接用all代表全部属性

Animations功能

transition的局限性

  1. transition是需要事件触发的，所以没法在网页加载时自动发生
  2. transition是一次性的，不能重复发生
  3. transition只能定义开始状态和结束状态，不能定义中间状态

于是出现了animations

  • @keyframes 定义动画，描述动画关键帧

  • 如果有多个变化 使用百分比定义每个时间点的样式

使用动画

  • animation-name: 指定动画的名称，@keyframes后面定义的名称
  • animation-duration: 指定完成整个动画要花费的时间
  • animation-timing-function: 指定动画过渡的方式 也可以使用cubic-bezier函数，默认是ease
  • animation-delay: 指定延迟多少秒开始执行动画
  • animation-iteration-count: 指定动画的执行次数，infinite表示无限次
  • animation-direction: 指定动画的执行方向，可指定的数如下
    • normal: 初始值（动画执行完毕后返回初始状态，默认值）
    • alternate: 交替更改动画的执行方向
    • reverse: 反方向执行动画
    • alternate-reverse: 从反方向开始交替更改动画执行方向
  • 结合使用 animate: name time funtion delay count direction

animation-direction的几个参数研究

  • normal 这个不用说，动画结束后，返回初试状态，算一次动画（count）

  • alternate 交替更改动画的执行方向，开始–结束–开始，算两次动画（count）

  • reserve 反方向执行动画 结束-开始 算一次动画（count）
  • alternate-reserve 反方向交替更改动画执行方向，结束–开始–结束，算两次动画

Web Animations API

CSS动画的局限性

  • 书写需要大量样式代码
  • 不能随心所欲的操纵动画的开始暂停回放

控制动画播放

  • pause() 动画暂停
  • play() 动画播放

取消动画、回放动画、设置动画速率
